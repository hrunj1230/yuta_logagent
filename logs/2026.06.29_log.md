## 학습 내용

### LangGraph 핵심 개념
- **LangGraph**: 상태를 가진 LLM 애플리케이션의 워크플로와 에이전트를 그래프 구조로 설계하고 실행하는 LangChain 팀의 저수준 오케스트레이션 프레임워크
- 실행 흐름을 미리 그래프로 설계하는 파이프라인 구조
- 기존 Chain(직선구조)과 달리 Graph(그래프 구조)로 복잡한 워크플로우 표현 가능

### 주요 구성 요소
**1. 상태 관리**
- **StateGraph**: 그래프를 만들어내는 빌더 클래스로, 그래프 실행 시점의 현재 스냅샷을 담는 공유 데이터 구조
- **MessagesState**: 메시지를 누적하는 LangGraph 기본 상태 스키마
- **Reducer**: 기존 값과 새 값을 합치는 로직을 담은 함수

**2. 그래프 구조**
- **Node**: 상태를 입력받아 작업을 수행하고 갱신할 부분을 출력하는 함수
- **Edge**: 노드 실행 순서를 표현하는 유향(directed) 그래프 연결로, 한 노드의 실행이 끝난 뒤 다음 노드 또는 종료 지점을 지정

**3. 도구 호출**
- **Tool Calling**: 사용자 입력 분석 결과로 도구 호출을 판단하는 기능
- **Tool Node**: LLM이 판단한 도구 선택을 바탕으로 실제 수행하여 결과 메시지를 만드는 사전 구성 노드

### 디자인 패턴

**1. Routing (라우팅)**
- 입력 또는 상태에 기반하여 다음 실행 노드를 선택하는 디자인 패턴
- LLM을 활용한 라우팅이 많이 사용됨
- 라우팅 함수를 조건부 엣지에 등록하거나, 커맨드 객체를 반환하는 함수를 노드로 등록

**2. Parallelization (병렬화)**
- 하나의 노드로부터 여러 노드에 연결되어 동시에 실행된 후 모든 노드의 결과를 취합하는 병렬 실행 패턴
- 직렬 처리가 느릴 때 정적 크기로 분할하여 병렬 처리
- **Send**: 런타임에 생성된 데이터 목록을 기준으로 동일한 노드를 여러 번 병렬 실행하도록 지시하는 동적 분기 객체

**3. Loop (반복)**
- 종료 조건이 만족될 때까지 반복하는 LangGraph 구조
- 무한루프 방지를 위한 종료 조건 설정 중요

**4. ReAct Pattern (추론-행동 패턴)**
- LLM이 추론(Reasoning)하고 실행한 다음 결과를 바탕으로 반복하는 에이전트 실행 구조
- 판단 + 행동을 반복하는 에이전트 패턴

### 기타 주요 개념
- **Command**: 상태 갱신과 다음 노드 지정을 할 수 있는 LangGraph 객체
- **Functional API**: 함수 호출 흐름을 워크플로우로 실행하는 API
- **Graph Visualization**: 그래프 구조 시각화

### 그래프 빌더 구축 순서
```
상태 스키마(class) → node(함수) → routing(함수)
그래프 빌더(스키마) → 노드 추가 → 엣지 추가 → 컴파일
```
